DEMO

En este demo de pasarela de pago se a utilizado Wompi y su metodo de integracion web checkout, el cual nos ofrece dos opciones, Una implementacion mediante Widget y Web checkout.


USO

  • Para una transacción de pruebas con tarjeta puedes usar los siguientes números de tarjeta para obtener respuestas distintas:

    • 4242 4242 4242 4242 para una transacción aprobada, Cualquier fecha de expiración en el futuro y CVC de 3 dígitos son válidos.

    • 4111 1111 1111 1111 para una transacción declinada, Cualquier fecha de expiración en el futuro y CVC de 3 dígitos son válidos.

  • Para una transaccion de pruebas con nequi debes tener en cuenta los siguientes números:

    • 3991111111 ara generar una transacción aprobada

    • 3992222222 para generar una transacción declinada

  • Para una transaccion de pruebas con PSE verás listados los siguientes bancos para tu elección:

    • Banco que aprueba: Con este, obtienes una transacción APROBADA de PSE.

    • Banco que rechaza: Con este, obtienes una transacción DECLINADA de PSE.
